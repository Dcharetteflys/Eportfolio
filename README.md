## **Enhancement One: Software Design and Engineering**
 The artifact I selected for the Software Design and Engineering category is a mobile application originally developed in CS-360: Mobile Architecture and Programming during the Fall 2025 semester. Created using Java in Android Studio, this project was the first mobile application I built and represents an important milestone in my growth as a software developer. I chose this artifact because it demonstrates my ability to work with technologies beyond Python and C++, while showcasing skills in mobile application development, software design, and problem-solving. During the enhancement process, I reviewed the original codebase and identified opportunities to improve both its readability and reliability. I added descriptive comments throughout the application to explain the purpose of key methods and improve clarity for anyone reviewing the code. I also corrected several syntax errors and resolved issues caused by code executing in an improper order, which led to inconsistent and seemingly random application errors. These improvements made the application more stable, easier to understand, and simpler to maintain.

 Enhancing this artifact taught me that developing software involves much more than creating a program that simply works. Revisiting the project allowed me to evaluate my earlier work with a more experienced perspective and address issues that I had not fully resolved during the original course due to time constraints. Troubleshooting syntax mistakes, tracing the source of unpredictable errors, and reorganizing portions of the code strengthened my debugging and analytical skills. It also reinforced the importance of testing, documentation, and thoughtful program design in producing professional-quality software. Through these enhancements, I demonstrated course outcomes related to software design, evaluating and improving existing solutions, debugging complex problems, and applying professional coding practices to create more reliable and maintainable applications.

## **Enhancement Three: Databases**

The artifact selected for the databases category was enhanced during the summer of 2025 as part of my ePortfolio. It is a Python application that uses PyMongo as the backend for an interactive database designed for animal rescue organizations. The original version implemented basic CRUD (create, read, update, and delete) operations that allowed users to manage records within the database.

I chose this artifact because it demonstrates my understanding of backend database development and how programming languages such as Python interact with stored data. During the enhancement process, I improved the readability and maintainability of the code by adding detailed notes and comments to explain the purpose and functionality of each section. These additions provide greater clarity for others reviewing the code. I also identified and repaired bugs that caused inconsistent behavior when updating and deleting records, improving the reliability of the application.

This experience reinforced the importance of writing clear, well-documented code and thoroughly testing software to identify and resolve issues. While the original CRUD functionality provided a working database application, refining the code demonstrated how improvements in documentation and reliability can significantly enhance the quality and usability of a project. It emphasized the value of creating software that is not only functional, but also maintainable and accessible to other developers.
